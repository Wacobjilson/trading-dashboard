# The Falsification Gate (Phase 20)

**A real edge is one that survives falsification.** In-sample performance is
never reported as an edge — if a strategy has only been tested in-sample, its
edge confidence is 0 by definition. The gate is allowed, and encouraged, to
conclude **"no validated edge."** Its integrity is measured by how many
plausible-looking candidates it correctly **rejects**.

## The five gates (every candidate produces a SURVIVAL REPORT)
Applied over the deep-history sector matrix (`_sector_matrix`, ~8yr Yahoo-
adjusted). Each family's trades are generated by its Zoo rule, then:

1. **Walk-forward OOS split** — trades sorted chronologically, first 60% train /
   last 40% test, strictly separated. Reporting in-sample as OOS is a *critical*
   verification failure (OOS-purity check asserts train-end ≤ test-start).
2. **Transaction cost + slippage** — a `%.2f%%`-style round-trip haircut
   (`FALSIFY_COST_RT` = 0.10%) is applied to **every** trade before any metric.
   No edge is reported without a cost model attached.
3. **Deflated Sharpe Ratio** (Bailey & López de Prado 2014) — the OOS per-trade
   Sharpe is compared to `SR0 = E[max Sharpe]` under the null across the trial
   count N (using the variance of the N families' Sharpes and the Euler-
   Mascheroni expected-maximum formula). `DSR = Φ((SR−SR0)·√(T−1) / √(1 −
   skew·SR + ((kurt−1)/4)·SR²))`. **Bar: DSR ≥ 0.95.** This is how best-of-N
   snooping is discounted — silent p-hacking is treated exactly like a
   mislabeled synthetic bar in Phase 18.
4. **Regime robustness** — each OOS trade is tagged by `regime_at` (bull/bear ×
   vol); a `validated` edge must be positive in **≥2 regimes**. One-regime
   edges are labeled regime-dependent, not validated.
5. **Random-entry permutation null** — 300 shuffles replace each entry with a
   random day of the same holding period and cost; the p-value is the fraction
   of null means ≥ the real mean. **Bar: p < 0.05** (the *signal* must beat
   random timing). Plus a stated **capacity** note.

## Verdict
`validated` ⇔ OOS-positive-after-costs AND DSR≥0.95 AND ≥2 positive regimes AND
perm p<0.05. Anything less is `candidate` (with the exact killer logged) or
`rejected`. Fundamental/flow families with no historical data (deep-value,
PEAD, GEX) are `untestable` — the engine **refuses to fake an OOS test** and
sets edge confidence 0 (deep-value is instead tracked forward-only, Phase 19).

## Provenance
Every report carries: `basis:"backtest"`, data source, test period, trial
count, cost model. Backtested is never presented as a track record (paper/live
labels enforced by verification).

## Why this is the capstone
Phase 18 made *numbers* prove themselves; Phase 19 made *fundamentals* prove
themselves; Phase 20 makes *strategies* prove themselves. Same zero-trust
standard, one level up.
